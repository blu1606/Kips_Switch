## 1. Password Hint Generator (Phase 9.2 + 9.4 + 9.7)
Tests: **Hint Generation**, **Anti-Doxxer (Secret Scrubbing)**, and **Semantic Caching**.

### ‚úÖ T1: Basic Hint Generation
**Goal:** Verify AI can generate a distinct hint from context.
```bash
curl -X POST http://localhost:3000/api/ai/generate-hint \
  -H "Content-Type: application/json" \
  -d '{
    "context": "The password is the name of the cafe where we met in Paris during the rainy summer of 2018.",
    "recipient": "Wife"
  }'
```
**Expected Output:** JSON with a clever riddle/hint (e.g., "The rainy spot in the city of lights...").

### üõ°Ô∏è T2: Anti-Doxxer Test (Safety)
**Goal:** Verify sensitive data (private keys) is scrubbed BEFORE reaching the AI.
```bash
curl -X POST http://localhost:3000/api/ai/generate-hint \
  -H "Content-Type: application/json" \
  -d '{
    "context": "My private key is 5KQwrPbwdL6PhXujxW37FSSQZ1JiwsST4cqQzDeyXtP79zkvFD3 and the password is banana.",
    "recipient": "Self"
  }'
```
**Expected Output:** Result should usually *not* contain the key. Check server logs for `[AI] Sensitive data detected`.

### ‚ö° T3: Semantic Cache Test
**Goal:** Verify repeated requests return `provider: "cache-hit"` and low latency.
1. Run T1 (Basic Hint) once.
2. Run it again (same or very similar text).
3. **Expected Output:** Second response should have `"provider": "cache-hit"` and `latency: ~150` (vs ~1000+ for fresh).
